动态规划 Dynamic Programming

动态规划在 Wiki 上的定义。

动态规划（英語：Dynamic programming，简称 DP）是一种在数学、管理科学、计算机科学、经济学和生物信息学中使用的，通过把原问题分解为相对简单的子问题的方式求解复杂问题的方法。
动态规划常常适用于有重叠子问题和最优子结构性质的问题，动态规划方法所耗时间往往远少于朴素解法。

动态规划的适用情况有三个适用情况的要求。

最优子结构性质。如果问题的最优解所包含的子问题的解也是最优的，我们就称该问题具有最优子结构性质（即满足最优化原理）。最优子结构性质为动态規劃算法解决问题提供了重要线索。

无后效性。即子问题的解一旦确定，就不再改变，不受在这之后、包含它的更大的问题的求解决策影响。

子问题重叠性质。子问题重叠性质是指在用递归算法自顶向下对问题进行求解时，每次产生的子问题并不总是新问题，有些子问题会被重复计算多次。动态規劃算法正是利用了这种子问题的重叠性质，对每一个子问题只计算一次，然后将其计算结果保存在一个表格中，当再次需要计算已经计算过的子问题时，只是在表格中简单地查看一下结果，从而获得较高的效率。

动态规划的思考方法

我认为动态规划的核心是 “把原问题分解为相对简单的子问题的方式求解复杂问题” 这一段话。其实许多算法或者解决问题的方案都体现了这个思想，比如分治算法，递归，递推等等。此类问题其实有着相似的思考流程。

1，定义问题

首先我们要定义想解决的问题。如何合适的定义问题是动态规划中最困难和需要灵机一动的地方，可以结合动态规划的常见套路模板学习。合适的问题定义可以减少思考难度，提高算法效率。这里结合一道简单的动态规划来理解定义问题。

Leetcode 198. 打家劫舍

你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警。

给定一个代表每个房屋存放金额的非负整数数组，计算你在不触动警报装置的情况下，能够偷窃到的最高金额。

示例1:

输入: [1,2,3,1]

输出: 4

解释: 偷窃 1 号房屋 (金额 = 1) ，然后偷窃 3 号房屋 (金额 = 3)。
偷窃到的最高金额 = 1 + 3 = 4 。

这道算法题我们可以定义要解决的问题为 对于前i所房屋，我们可以偷取的最大现金数 。
用函数表示即为 getMaxMoney(int i) ，i为前i所房屋。或者可以用数组表示为dp[i]。

2，找到子问题和问题的关系

定义问题和找子问题即为许多动态规划教程提到的找状态转移方程，关键是对于我们定义的问题，找到子问题和母问题的关系来简化问题的求解，其实思考过程和递归，分支问题十分类似。高中数学中的数学归纳法，递推式也是类似的思考方向。

同样以 Leetcode 198 作为例子。 我们已经有了问题的函数表示 getMaxMoney(int i) 。
观察问题，我们发现我们不可以选择两间相邻的房屋，因此我们的问题可以分解成一个选择，两个方向。

选择 ：是否抢劫房屋 i 。

方向 1 : 如果不抢劫房屋 i， 那么我们可以抢劫房屋 i - 1。在这个方向下，getMaxMoney(i) = getMaxMoney(i - 1) 。

方向 2 ： 如果抢劫房屋i， 那么我们不可以抢劫房屋 i - 1。在这个方向下，getMaxMoney(i) = getMaxMoney(i - 2) + money[i]。

当我们弄清楚了我们的选择和结果之后，我们需要的就是比较哪种结果更好，即为选取两种结果的最大值。到此为止，我们就找到了子问题和问题的关系。

3，记忆化

记忆化是动态规划算法效率高于普通递归解的关键，他的核心思想是 空间换时间， 通过记录已经计算过的问题来减少重复计算，提高效率。通常我们使用数组或者 Map 来进行记忆。

对于抢劫问题，我们已经定义了问题为 getMaxMoney(int i) 。现在我们定义一个数组 dp[i]， 表示和 getMaxMoney(int i) 一样的结果。

现在我们的问题变成了，如何求出 dp[n]，n 为最后一间房屋。 考虑我们有一张表格

House Num Money getMaxMoney
0 1 x
1 2 x
2 3 x
3 1 x

我们要求的即为最后一个格子的值。 那么我们就有两种填格子的方法，一种是自上而下，另一种是自下而上。注意这里的自上而下和自下而上和表格的上下左右没有关系，而是对两种不同方法的描述。

4， 自上而下 Top - Down

自上而下解法即为通过递归调用，从最大的问题出发，找他们的子问题是否计算过。如果计算过，直接返回结果，如果没有计算过，则递归向下解决。

自上而下解法的好处在于我们不需要思考需要按照什么顺序来填格子。另外当我们得到最后一个格子的时候，这个表格中有一些值可能仍然是空的，这样我们就节省了计算这些不必要结果的时间。

5，自下而上 Bottom - Up

自下而上的方法即为通过循环，从子问题出发，一步一步踏踏实实的填满要求的格子。自下而上的优缺点就是自上而下的优缺点翻转过来。在自下而上解法中，通常我们定义 dp 数组来记忆化。

自下而上方法的关键在于，我们要额外的思考计算的顺序，尤其是对于多维 dp 问题。以抢劫犯问题距离，我们是从左往右，先算 i 小的时候的值，还是从右往左，计算 i 大的时候的值？ 对于抢劫犯问题，答案是显而易见的。

如果无法显而易见的想到顺序，那么我们需要画表来解决，表的维度即为dp的维度。在表中随意选择某个需要计算的值，观察计算该值需要哪些值。如果需要左边和上边的值来求解，那么即为从左向右，从上向下遍历。同时通过观察表可以得出遍历的区间。

6，基本值 Base Case

无论是自上而下解法，还是自下而上解法，我们都需要对基本值进行讨论。基本值即为最小问题，该问题无法划分为更小的问题，在代码里通常会表现为越界的负数 index 对数组访问。

以抢劫犯问题为例，当 i = 0 和 i = 1 的时候，我们发现我们需要 i = -1 和 -2 的值。对于 -1 和 -2， 我们的问题是没有定义的。因此， 我们需要直接给 dp[0] 和 dp[1] 进行赋值。基本值通常非常容易观察得到。

7， 时间复杂度分析

动态规划的时间复杂度分析分为两部分。

首先，我们需要考虑填充的格子数，即为dp维度。假设 input size 为n， 通常一维dp是 O(n)， 二维dp是 O(n^2)， 以此类推。

其次，我们需要考虑计算每一个格子所需要的时间。 我们需要观察状态转移方程来得到这部分的时间复杂度。 抢劫犯问题该部分时间为 O(1)。

得到两部分的结果之后，最后结果为时间复杂度相乘。 因此对于抢劫犯问题，时间复杂度为 O(n)。
